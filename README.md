# Roast-Lens 표준 측정 프로토콜

Roast-Lens는 휴대폰 카메라로 분쇄 원두의 CIE Lab 색상을 계산하고 Agtron 범위를 추정하는 비교용 도구입니다. 표시되는 Agtron 값은 CIE L* 기반 앱 추정치이며 인증 계측기 결과를 대체하지 않습니다.

## 가이드선 기준

- 바깥 실선: 종이컵 입구 전체를 맞추는 기준입니다.
- 안쪽 점선: 앱이 실제 색상을 채취하는 고정 영역입니다.
- 원두 한 알, 원두 더미의 가장자리 또는 컵 바닥에 선을 맞추지 않습니다.
- 카메라 높이를 센티미터로 맞추지 않습니다. 바깥 실선과 컵 테두리가 일치하도록 높이를 조절합니다. 이렇게 해야 화각이 다른 휴대폰에서도 컵이 같은 비율로 촬영됩니다.

## 표준 시료

- 컵: 동일 제품의 180mL 무광·무인쇄 흰 종이컵
- 시료: 중간 분쇄한 원두 20.0g
- 평탄화: 누르거나 다지지 않고 컵을 좌우로 가볍게 3회 흔듭니다.
- 조명: 직사광선과 강한 외부 반사를 피하고, 가능하면 휴대폰 플래시를 켭니다.
- 기기: 비교 시험 한 묶음은 동일한 휴대폰과 동일한 브라우저를 사용합니다.

위 규격은 Roast-Lens 내부 비교를 위한 앱 전용 프로토콜입니다. 공식 SCA 또는 Agtron 표준 시료 규격을 의미하지 않습니다.

## 측정 절차

1. 같은 그라인더 설정으로 시료를 중간 분쇄합니다.
2. 20.0g을 같은 규격의 흰 종이컵에 담고 3회 가볍게 흔들어 평탄화합니다.
3. 카메라를 켜고 바깥 실선을 컵 입구 전체에 맞춥니다.
4. 십자선이 컵 중심에 오고, 컵 테두리가 기울어진 타원이 아니라 원형에 가깝게 보이도록 휴대폰을 평행하게 유지합니다.
5. 안쪽 점선이 분쇄 원두로 빈틈없이 채워졌는지 확인합니다.
6. 컵 감지, 중앙, 거리, 수평, 채움이 모두 통과해 `READY`가 표시되면 `MEASURE`를 누릅니다.
7. `PASS` 결과만 사용합니다. 컵을 120도씩 돌려 총 3회 측정하고 Agtron 추정치의 중앙값을 기록합니다.

## 앱의 자동 합격 기준

- 컵 테두리 감지: 48방향 중 54% 이상에서 밝고 무채색인 테두리 확인
- 중앙: 화면 중심과 추정 컵 중심의 오차 4.5% 이하
- 거리: 추정 컵 반경과 가이드 반경의 오차 10% 이하
- 수평: 가로/세로 비율 오차 12% 이하, 테두리 반경 변동 14% 이하
- 채움: 안쪽 측정 영역의 86% 이상이 흰 바닥이 아닌 시료
- 촬영 유지: 30프레임 중 75% 이상이 위 조건을 계속 통과

기준 미달 프레임은 최종 색상 계산에서 제외합니다. 촬영 중 합격 프레임이 75% 미만이면 결과를 저장하지 않고 재측정을 요청합니다.

## 분쇄도 측정 절차

### 100원 기준 모드

1. 무광 흰 종이 위 왼쪽 가이드에 현용 100원 주화를 놓습니다. 지름 24.00mm를 기준값으로 사용합니다.
2. 동전과 분쇄 원두가 같은 바닥면에 닿도록 놓습니다.
3. 원두를 중앙 점선 안에 겹치지 않게 한 겹으로 펼칩니다.
4. 동전 가이드가 초록색이고 `READY`가 표시되면 패널의 `분쇄도 측정` 버튼을 누릅니다.
5. 앱은 5프레임을 측정하고 프레임 간 D50 편차가 12%를 넘으면 결과를 거부합니다.

### 동전 없이 예측 모드

1. `동전 없이`를 선택하고 원두를 중앙 점선 안에 한 겹으로 펼칩니다.
2. 비교 측정은 같은 휴대폰과 같은 카메라 높이를 유지합니다.
3. 앱은 정사각형 화면 전체 폭을 160mm, 중앙 분석 영역을 56mm로 가정해 입자 크기를 예측합니다.
4. 결과에는 `~`와 `ESTIMATE`가 표시되며 신뢰도는 최대 `MID`로 제한됩니다.

분쇄도 값은 입자의 투영면적 등가지름입니다. 100원 기준 모드는 동전으로 실제 비율을 환산하지만, 동전 없는 모드는 물리 기준이 없는 상대 비교용 예측치입니다. D10, D50, D90은 투영면적 가중 분포이며 인증된 체 분석기나 레이저 입도분석기의 결과를 대체하지 않습니다.

## 결과 해석

- `PASS`: 위치, 거리, 수평, 컵 기준색, 시료 채움이 앱 기준을 통과했습니다.
- `HIGH / MID / LOW`: 합격 프레임 비율, 흰 컵 기준색, 프레임 간 L* 흔들림을 합친 측정 신뢰도입니다.
- 기기 간 비교: 흰 컵 보정으로 차이를 줄이지만 카메라 센서, 렌즈, 자동 노출 처리 차이는 완전히 제거할 수 없습니다.
- 색상 보정: 흰 컵의 RGB 기준값은 선형광 sRGB 공간에서 채널별 보정하고, 흰 기준을 유지하면서 원두의 어두운 중간톤을 소폭 보완합니다. 합격 프레임의 극단 밝기값은 제외합니다.
- 절대값 판정: 제품 규격 판정이나 품질 인증에는 교정된 색차계 또는 Agtron 계측기를 사용해야 합니다.

## 시험 기록 권장 항목

날짜, 시료명, 로스팅 배치, 분쇄 설정, 컵 제품, 휴대폰 모델, 브라우저, 3회 L*, 3회 Agtron 추정치, 중앙값, 신뢰도, 비고를 함께 기록합니다.

## GitHub Pages 배포

배포 진입 파일은 저장소 최상위의 `index.html`입니다. 현재 폴더에서는 다음 세 파일만 GitHub 저장소 루트에 올리면 됩니다.

- `index.html`
- `README.md`
- `.nojekyll`

Git과 GitHub CLI가 설치되어 있지 않은 PC에서는 GitHub 웹사이트에서 새 저장소를 만든 뒤 `Add file` → `Upload files`로 위 파일을 업로드할 수 있습니다.

1. 저장소의 `Settings` → `Pages`로 이동합니다.
2. `Build and deployment`의 `Source`를 `Deploy from a branch`로 선택합니다.
3. Branch는 `main`, 폴더는 `/(root)`를 선택하고 저장합니다.
4. 배포가 끝나면 같은 Pages 화면에서 공개 URL을 확인합니다.
5. 카메라 사용을 위해 `Enforce HTTPS`가 활성화되어 있는지 확인합니다.

GitHub Free 계정이라면 공개 저장소 사용을 권장합니다. 저장소에 측정 기록이나 개인정보를 넣지 말고 정적 앱 파일만 배포합니다.

## 배포 후 확인

1. 공개 URL이 `https://`로 열리는지 확인합니다.
2. Android Chrome 또는 iPhone Safari에서 카메라 허용과 `다시 시도`가 동작하는지 확인합니다.
3. `COLOR`, `GRIND SIZE`, `100원 기준`, `동전 없이` 전환 후 화면이 겹치지 않는지 확인합니다.
4. 실제 시료는 같은 조건으로 3회 측정하고 중앙값을 비교합니다.

카메라 프레임과 측정 결과는 브라우저 안에서만 처리하며 서버로 전송하지 않습니다. 화면 표시를 위한 외부 글꼴 리소스만 불러옵니다.
